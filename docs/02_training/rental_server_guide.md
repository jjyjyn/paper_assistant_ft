# 租服务器实操指南（AutoDL优先）

## 结论先说

- 你的项目目标（Qwen 中文任务 + LoRA SFT）用单卡 `RTX 4090 24GB` 是够用的。
- 学生价约 `1.8 元/小时`（以你当前看到的价格为准，实际以下单页面实时价格为准）。
- 如果节点稳定，首轮可用训练通常可在 `2-6 小时` 内完成（取决于数据量和超参）。

## 建议租什么型号

### 首选：RTX 4090 24GB（性价比最高）

- 适用：Qwen2.5-3B / Qwen3-4B 的 LoRA 微调
- 优点：速度快、价格低、24GB 显存够做 SFT
- 适合你当前阶段：快速做 first usable version + resume-ready version

### 备选：A5000/A40（24/48GB）

- 适用：你对显存稳定性或长作业稳定性有更高要求
- 缺点：通常价格高于 4090，性价比不一定更好

### 不建议当前就上 A100/H800

- 你的当前任务不是大规模多卡预训练
- 成本会显著增加，不符合“最快完成简历闭环”目标

## 4090 对本项目够不够

- 够。对 `Qwen2.5-3B-Instruct` 做 LoRA 非常稳。
- `Qwen3-4B-Instruct` 也能尝试，但建议先用 Qwen2.5-3B 跑通全链路，再上 Qwen3。
- 推荐先跑通再追求更大 batch 和更长训练。

## 训练时间估算（单卡 4090）

以下是实用估算区间（便于预算）：

- 数据量 100-300 条（v0 高质量小集）：
  - 单轮训练：约 `0.5-2 小时`
- 数据量 1k-3k 条：
  - 单轮训练：约 `2-6 小时`
- 数据量 5k+：
  - 单轮训练：约 `6-12+ 小时`

说明：训练时长受 `max_len`、`batch_size`、`grad_accum`、`epoch` 影响很大。

## 数据上传时长估算

粗略公式：`上传时间 ≈ 文件大小 / 实际上传速率`

- 1GB 数据：
  - 10MB/s：约 2 分钟
  - 5MB/s：约 4 分钟
- 10GB 数据：
  - 10MB/s：约 17 分钟
  - 5MB/s：约 34 分钟
- 20GB 数据：
  - 10MB/s：约 34 分钟
  - 5MB/s：约 68 分钟

实践建议：
- 小文件先打包（`zip`/`tar`）再传
- 代码用 git，同步快；模型/数据用网盘或文件存储

## AutoDL 选机与开机检查清单

1. 选 `4090 24GB`，优先选择网络评价好的地区。
2. 优先选“可扩容硬盘”更大的实例，避免中途扩盘中断。
3. 开机后先检查：
   - `nvidia-smi`
   - `python -c "import torch; print(torch.cuda.is_available())"`
4. 先跑 10 分钟最小训练冒烟，再开长任务。

## 成本预算参考

按 `1.8 元/小时` 估算：

- 3 小时：`5.4 元`
- 6 小时：`10.8 元`
- 10 小时：`18 元`
- 20 小时：`36 元`

建议预算：
- first usable version：`10-30 元`
- resume-ready version（含多轮对比）：`30-80 元`

## 迁移策略（从当前服务器到租用服务器）

1. 本地仓库保持主线，继续用 git 管理代码。
2. 新服务器只做执行环境，不改工作流。
3. 迁移顺序：
   - 拉代码
   - 建环境
   - 上传数据
   - 下载/挂载模型
   - 先跑冒烟训练再开正式训练

## 明确决策建议

- 如果你明天节点仍不稳，直接租 `4090 24GB` 开工。
- 模型优先顺序：`Qwen2.5-3B-Instruct -> Qwen3-4B-Instruct`。
- 目标优先顺序：先拿到可展示闭环，再优化指标。
