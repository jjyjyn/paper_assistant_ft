# 面试与答辩笔记（Interview Notes）

## Why：核心决策

### 为什么坚持 LLaMA-Factory + LoRA

- 目标是 5-7 天交付可讲清楚的闭环，优先工程可执行与可复现
- LLaMA-Factory 训练/评测/导出链路完整
- LoRA 适配单卡 24GB，迭代快、成本低

### 为什么是 Qwen 路线

- 任务是中文论文精读与实验解析，Qwen 系列中文表现和指令跟随能力匹配
- 先跑通 Qwen2.5-3B，再择机切 Qwen3-4B，属于工程上的分阶段推进，不是方向改变

### 为什么今天优先记录体系

- 不是只“跑通训练”，还要能复盘、能答辩、能写简历
- 文档化能沉淀：
  - 做了什么
  - 为什么这么做
  - 碰到问题怎么解决

## Day 1 可能被问的问题

1. 为什么 Day 1 在网络/环境上花了很多时间？
- 回答：训练项目前置风险最大的是环境不可复现，先打通链路可避免后续返工。

2. 为什么你说 Day 1 完成了？
- 回答：代码同步、环境创建、核心依赖、CLI 验证都已完成；剩余是节点稳定性问题，不是路线失败。

3. 为什么不直接放弃这条路线？
- 回答：路线不变，执行策略分层。先完成数据与配置，再在稳定 GPU 上开训，整体效率更高。

## 关于租服务器的答辩话术

1. 为什么选 4090（而不是 A100）？
- 回答：当前目标是 LoRA SFT 闭环，不是大规模预训练。4090 的性价比对单卡微调最好，足够支撑 3B-4B 级别模型。

2. 你怎么估算训练成本和时长？
- 回答：按数据规模和超参做区间估算，小规模 v0 数据通常 0.5-2 小时可跑完一轮；按小时计费可控，先验证再扩量。

3. 如果学校服务器不稳定怎么办？
- 回答：项目流程已拆分为“数据/配置”和“训练执行”两部分。前者可先完成，后者平迁到租用节点，不影响总体交付。
