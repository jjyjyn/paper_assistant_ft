# 面试与答辩笔记（Interview Notes）

## 路线选择背后的 Why

### 为什么选 LLaMA-Factory

- 上手快：训练、评测、导出链路完整，适合 5-7 天冲刺闭环
- 社区成熟：常见报错和配置有大量参考，降低踩坑成本
- 工程可复用：后续可平移到其他中文任务或模型

### 为什么选 Qwen3-4B-Instruct

- 中文能力较强，和“论文精读/实验解析”任务语域匹配
- 4B 规模适合 24GB 显存做 LoRA SFT，训练成本可控
- Instruct 基座在问答、解释、追问场景更容易收敛

### 为什么选 LoRA SFT

- 参数高效，显存压力小，迭代快
- 对简历项目足够：能体现数据设计、训练调参、评测闭环能力
- 可解释性强：可以清晰说明超参与效果变化

### 为什么按四类任务组织数据

- 覆盖真实使用场景：贡献提取、方法对比、实验解析、追问
- 利于面试讲故事：可展示“能力矩阵”而非单点问答
- 便于后续评测拆分，定位薄弱环节

## 阶段 1（环境与仓库跑通）可能被问

### 老师/面试官可能问什么

1. 为什么不直接全参数微调？
2. 24GB 显存为什么选 4B 而不是更大模型？
3. 为什么要先做文档记录再开训？

### 你可以怎么回答

1. 全参数微调资源和时间成本更高，不符合 5-7 天闭环目标；LoRA 更适合快速验证和多轮迭代。
2. 4B + LoRA 在显存、速度、效果之间平衡更好，能保证稳定跑通并留出调参与评测预算。
3. 记录体系能把“做了什么 + 为什么这么做 + 遇到什么问题”固化，后续写简历和答辩更高效，也便于复现实验。
