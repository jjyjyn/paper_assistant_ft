# 面试与答辩笔记（Interview Notes）

## Why：今天的数据方案为什么这么设计

### 为什么先做“结构化案例库 + 模板扩展”

- 目标是快速形成可训练数据闭环，避免在环境波动期空耗时间。
- 用 20 条结构化案例可稳定扩展到 80 条四任务样本，覆盖面可控。
- 模板方法便于复现和迭代，后续可以按同样框架持续加样本。

### 为什么要四类任务等比例覆盖

- 项目目标不是单一问答，而是“贡献提取 + 对比解释 + 实验解析 + 追问答辩”完整能力。
- 等比例分布可避免模型偏科，训练和评测更容易定位短板。

### 为什么先做 smoke 训练再正式训练

- 降低算力浪费，先验证配置、数据路径、日志链路是否正确。
- 先看显存和吞吐，再调 batch/accum，正式训练更稳。

## Day 2 可能被问

1. 你的数据是不是“模板化太重”？
- 回答：v1 目的是最短时间拿到可用闭环；v2 会补人工高难样本和边界样本，模板是启动手段，不是终态。

2. 为什么不是直接人工写 1000 条？
- 回答：时间和成本不匹配当前阶段目标。先做可复现的中小规模高覆盖集，再按误差驱动扩容更高效。

3. 你如何保证数据质量？
- 回答：通过字段校验、任务分布校验、去重和抽样人工检查；并记录 source_case_id 保障可追溯。

## 关于租 4090 的答辩话术

1. 为什么选 4090？
- 回答：对 3B-4B LoRA 微调足够，单小时成本低，最符合“快速闭环”的工程目标。

2. 你如何控制成本？
- 回答：先本地完成数据与脚本，只在训练窗口租机；先 smoke 再长训，按效果决定是否继续投入时长。

3. 如果训练失败怎么办？
- 回答：先回放日志定位问题，修复后复用同一配置快速重启；数据和脚本已固化，迁移成本低。

## Phase 3 训练阶段常见追问

1. 你如何证明训练流程是可复现的？
- 回答：我把流程固定为 `server init -> data check -> smoke -> full`，并把模型路径、关键超参、日志目录、checkpoint 路径写入文档；同样命令可在新机器复现。

2. 为什么要同时准备 in-domain 和 external 两套测试？
- 回答：只看 in-domain 容易高估效果；external 能检验跨分布泛化。两套同时报告，结论更稳健。

3. 你怎么控制 GPU 成本？
- 回答：先做短时 smoke，确认链路和显存后再开 full；每次只改少量超参并记录效果，避免反复长训试错。
